## Agent Communication Protocol

Agents communicate through:

1. **GitHub Issues** - Task assignment and tracking
2. **PR Comments** - Code review and feedback
3. **Telemetry System** - Event logging and coordination
4. **Agent Registry** - Capability discovery and status
5. **Traffic Light System** - Deployment coordination

## Extending the Agent Team

To add a new agent:

1. Edit `.blackroad/agent-config.yml`
2. Add agent definition with capabilities
3. Register agent: `./blackroad-codex-integration.sh register NAME TYPE CAPABILITIES`
4. Update this manifest
5. Configure coordination rules

## Agent Best Practices

### For Users

- **Be Specific** - Tag the right agent for the job
- **Provide Context** - Give agents all necessary information
- **Use Templates** - Leverage issue/PR templates
- **Follow Coordination** - Respect agent coordination phases

### For Agents

- **Log Actions** - Use telemetry for all significant actions
- **Coordinate** - Follow coordination rules
- **Document** - Maintain clear documentation
- **Monitor** - Track system health and performance

## Agent Ethics

All BlackRoad agents follow these principles:

1. **Transparency** - All actions are logged and traceable
2. **Consent** - Never act without proper authorization
3. **Privacy** - Protect sensitive information
4. **Quality** - Maintain high standards
5. **Collaboration** - Work together effectively
6. **Autonomy** - Respect individual agent capabilities
